# Veesker Phase 4 — Cloud Auth Implementation Design

> **For agentic workers:** Use `superpowers:writing-plans` to turn this spec into an implementation plan, then `superpowers:subagent-driven-development` to execute phase by phase.

**Date:** 2026-04-28
**Repos:** `veesker` (desktop) + `veesker-cloud` (backend)

---

## Principle

Magic link authentication with polling. User enters email in the desktop app, receives a link by email, clicks it in the browser, and the desktop detects authentication automatically via polling. No deep link plumbing, works on all platforms.

**What this enables:** `FEATURES.cloudAI` and other Cloud flags become `true` after login. `CloudProvider.ts` (currently a stub) becomes a real HTTP client calling `api.veesker.cloud/v1/ai/chat` with the user's JWT.

---

## Architecture

```
Desktop                           Backend (Railway + Bun/Hono/Postgres)
──────────────────────────────    ────────────────────────────────────
LoginModal
  │  POST /v1/auth/magic-link/send
  │  { email, session_id }  ──────►  creates magic_link_sessions (pending)
  │                                  sends email via Resend
  │
  │  poll GET /v1/auth/poll/{session_id} every 3s (max 5min)
  │                          ──────►  { status: "pending" | "expired" | "authenticated" }
  │
  │  (user clicks link in browser)
  │         GET /v1/auth/magic-link/verify?token=xxx&session_id=yyy
  │                          ──────►  validates token, issues JWT,
  │                                   marks session as authenticated,
  │                                   redirects to app.veesker.cloud/auth/done
  │
  │  poll returns { status: "authenticated", token: JWT, features: {...} }
  ▼
  → store JWT in OS keyring via Tauri command
  → store features in localStorage
  → applyFeatureFlags(features)
  → close modal

Subsequent startup
  → read JWT from keyring (offline OK)
  → decode expiry locally
  → apply features from localStorage
  → GET /v1/auth/me in background (refresh features if tier changed)
```

## Backend

### New migration: `002_magic_link.sql`

```sql
CREATE TABLE IF NOT EXISTS magic_link_sessions (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id  TEXT NOT NULL UNIQUE,   -- UUID generated by desktop
  email       TEXT NOT NULL,
  token_hash  TEXT NOT NULL,          -- SHA-256 of the email token
  status      TEXT NOT NULL DEFAULT 'pending'
              CHECK (status IN ('pending', 'authenticated', 'expired')),
  jwt         TEXT,                   -- populated on verify
  features    JSONB,                  -- feature flags computed from org.tier
  expires_at  TIMESTAMPTZ NOT NULL,   -- 15 minutes from creation
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS magic_link_session_id ON magic_link_sessions (session_id);
CREATE INDEX IF NOT EXISTS magic_link_expires ON magic_link_sessions (expires_at)
  WHERE status = 'pending';
```

### Modified file: `src/routes/auth.ts`

Four new endpoints added alongside the existing `/stub-login`:

#### `POST /v1/auth/magic-link/send`

```
Body: { email: string, session_id: string (UUID) }

1. Validate body with zod
2. token = crypto.randomUUID()
3. token_hash = hex(SHA-256(token))
4. Upsert user: INSERT INTO users (org_id, email) ON CONFLICT (org_id, email) DO NOTHING
   - org_id = default org (SELECT id FROM organizations WHERE slug = 'default' LIMIT 1)
   - Create default org if not exists: INSERT INTO organizations (name, slug, tier) VALUES ('Default', 'default', 'personal') ON CONFLICT DO NOTHING
5. INSERT INTO magic_link_sessions (session_id, email, token_hash, expires_at = NOW()+15min)
6. sendMagicLink(email, `https://api.veesker.cloud/v1/auth/magic-link/verify?token=${token}&session_id=${session_id}`)
7. Return 200 { ok: true }

Always return 200 even if email doesn't exist (don't leak whether email is registered).
Return 503 if Resend fails.
```

#### `GET /v1/auth/magic-link/verify`

```
Query: token, session_id

1. SELECT * FROM magic_link_sessions WHERE session_id = $session_id
2. If not found or expired (expires_at < NOW()) or status != 'pending':
   redirect to app.veesker.cloud/auth/error?reason=invalid
3. Verify token_hash == hex(SHA-256(token)); if mismatch:
   redirect to app.veesker.cloud/auth/error?reason=invalid
4. Fetch user: SELECT u.id, u.role, o.tier FROM users u JOIN organizations o ON o.id = u.org_id WHERE u.email = $email
5. Compute features from tier (see Feature Flags section)
6. Issue JWT: issueToken({ sub: user.id, org: org.id, email, role }, "30d")
7. UPDATE magic_link_sessions SET status='authenticated', jwt=$jwt, features=$features WHERE session_id=$session_id
8. Redirect to app.veesker.cloud/auth/done
```

#### `GET /v1/auth/poll/:session_id`

```
1. SELECT status, jwt, features FROM magic_link_sessions WHERE session_id = $session_id
2. If not found: 404 { error: "not_found" }
3. If status = 'pending' and expires_at < NOW(): UPDATE status='expired', return { status: "expired" }
4. Return { status } — or { status: "authenticated", token: jwt, features } when done
```

#### `GET /v1/auth/me`

Auth-protected (JWT required). Returns current user info and features.

```
1. authMiddleware (existing) validates Bearer JWT
2. Fetch user + org tier from DB
3. Compute features from tier
4. Return { user: { id, email, role }, org: { id, slug, tier }, features }
```

## Desktop

### New file: `src/lib/services/auth.ts`

Startup auth initializer. Called once from the root layout on app boot.

```typescript
export async function initAuth(): Promise<void>
```

Flow:
1. `invoke("auth_get_token")` → `null` or JWT string
2. If null: return (stays CE)
3. Decode JWT payload (base64, no signature check): read `exp`
4. If `Date.now()/1000 > exp`: `invoke("auth_clear_token")`, return
5. `features = localStorage.getItem("veesker:features")`
6. If features: `applyFeatureFlags(JSON.parse(features))`
7. Background: `fetch("https://api.veesker.cloud/v1/auth/me", { headers: { Authorization: "Bearer " + token } })`
   - 200: `applyFeatureFlags(res.features)`, update localStorage
   - 401: `invoke("auth_clear_token")`, `resetFeatures()`
   - Network error: silent (offline — already applied from localStorage)

### Modified file: `src/lib/workspace/LoginModal.svelte`

Three reactive states:

```svelte
type State = "idle" | "waiting" | "error"
let state = $state<State>("idle")
let email = $state("")
let errorMessage = $state("")
```

**idle state:** Email input + "Enviar link" button.

**waiting state:** "Verifique seu e-mail" message + spinner + auto-starts poll loop.

Poll loop (inside the component):
```typescript
async function startPoll(sessionId: string) {
  const deadline = Date.now() + 5 * 60 * 1000;
  while (Date.now() < deadline) {
    await sleep(3000);
    const res = await fetch(`https://api.veesker.cloud/v1/auth/poll/${sessionId}`);
    const data = await res.json();
    if (data.status === "authenticated") {
      await invoke("auth_set_token", { token: data.token });
      localStorage.setItem("veesker:features", JSON.stringify(data.features));
      applyFeatureFlags(data.features);
      open = false;  // close modal
      return;
    }
    if (data.status === "expired") {
      state = "error";
      errorMessage = "Link expirado. Tente novamente.";
      return;
    }
  }
  state = "error";
  errorMessage = "Tempo esgotado. Tente novamente.";
}
```

Send flow:
```typescript
async function sendLink() {
  const sessionId = crypto.randomUUID();
  const res = await fetch("https://api.veesker.cloud/v1/auth/magic-link/send", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ email, session_id: sessionId }),
  });
  if (!res.ok) {
    state = "error";
    errorMessage = "Erro ao enviar e-mail. Tente novamente.";
    return;
  }
  state = "waiting";
  startPoll(sessionId);
}
```

**error state:** Error message + "Tentar novamente" button (resets to idle).

### Modified file: `src/lib/ai/providers/CloudProvider.ts`

Real implementation replacing the stub:

```typescript
export function CloudProvider(): AIProvider {
  return {
    async chat(params: ChatParams) {
      const token = await invoke<string | null>("auth_get_token");
      if (!token) {
        return { ok: false, error: { code: "UNAUTHORIZED", message: "Not logged in to Veesker Cloud." } };
      }
      const res = await fetch("https://api.veesker.cloud/v1/ai/chat", {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
          "Authorization": `Bearer ${token}`,
        },
        body: JSON.stringify({
          messages: params.messages,
          context: params.context,
        }),
      });
      if (res.status === 401) return { ok: false, error: { code: "UNAUTHORIZED", message: "Session expired. Please sign in again." } };
      if (res.status === 402) return { ok: false, error: { code: "PAYMENT_REQUIRED", message: "Credit limit reached. Visit veesker.cloud to top up." } };
      if (!res.ok)            return { ok: false, error: { code: "CLOUD_UNAVAILABLE", message: "Veesker Cloud is temporarily unavailable." } };
      const data = await res.json();
      return { ok: true, data };
    },
  };
}
```

Note: `POST /v1/ai/chat` on the backend is **out of scope for this spec** — it will be designed separately. `CloudProvider` returns `CLOUD_UNAVAILABLE` until that endpoint exists, which triggers BYOK fallback via `AIService`.

## Logout

`resetFeatures()` + `invoke("auth_clear_token")` + `localStorage.removeItem("veesker:features")`.

No dedicated endpoint needed — JWT is stateless; clearing the local token is sufficient for v1.

## Security notes

- Token in e-mail is a raw UUID; only the SHA-256 hash is stored in the DB.
- Sessions expire in 15 minutes and are single-use (status changes to `authenticated` on first verify).
- JWT expiry is 30 days; `/v1/auth/me` refreshes features but does not renew the JWT (user re-authenticates after 30 days).
- `session_id` is desktop-generated (UUID v4) — no server-side secret needed for polling.
- Credentials, wallets, and TNS strings never leave the user's machine.

## Testing

### Backend (Bun test)

- `POST /v1/auth/magic-link/send` with valid body → 200, session created in DB
- `GET /v1/auth/poll/{session_id}` pending → `{ status: "pending" }`
- `GET /v1/auth/magic-link/verify` valid token → session becomes authenticated, JWT issued
- `GET /v1/auth/poll/{session_id}` after verify → `{ status: "authenticated", token, features }`
- `GET /v1/auth/magic-link/verify` expired session → redirect to error page
- `GET /v1/auth/magic-link/verify` wrong token → redirect to error page
- `GET /v1/auth/me` valid JWT → 200 with features
- `GET /v1/auth/me` no token → 401
- `computeFeatures("personal")` → teamFeatures false
- `computeFeatures("business")` → teamFeatures true

### Desktop (Vitest)

- `auth.ts initAuth()`: null token → features unchanged (CE)
- `auth.ts initAuth()`: expired JWT → token cleared, CE
- `auth.ts initAuth()`: valid JWT offline → features from localStorage applied
- `CloudProvider` no token → `{ ok: false, error: { code: "UNAUTHORIZED" } }`
- `CloudProvider` 401 response → `{ ok: false, error: { code: "UNAUTHORIZED" } }`
- `CloudProvider` 402 response → `{ ok: false, error: { code: "PAYMENT_REQUIRED" } }`
- `CloudProvider` 500 response → `{ ok: false, error: { code: "CLOUD_UNAVAILABLE" } }`
