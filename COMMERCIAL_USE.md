# Commercial Use & Licensing

> **Status: aspirational framework.** Veesker is in early access. Until commercial subscriptions launch, the packaged app is free for all uses including commercial. This document describes the framework that will activate when billing goes live.

Veesker is licensed under the [Apache License 2.0](LICENSE), which permits free commercial use of the source code. **However**, the maintainer offers Veesker as a packaged desktop application under a separate commercial use policy described below.

This is the same model used by Docker Desktop and similar tools — the source code is fully open, but using the **packaged desktop application** in larger organizations requires a paid subscription.

## Free for

You may use the Veesker desktop application **for free** if any of the following apply:

- **Personal use** — students, hobbyists, individual learning
- **Open-source projects** — building or maintaining open-source software
- **Education** — teaching, research, or non-commercial coursework
- **Small companies** — organizations with **fewer than 50 employees** AND **less than US$ 5 million** in annual revenue
- **Government / non-profit (small)** — small public-sector or non-profit organizations meeting the same headcount/revenue threshold

If you fall in any of these categories, you can use Veesker indefinitely at no cost.

## Paid subscription required for

Use of the **packaged Veesker desktop application** in a commercial setting requires a paid subscription if your organization has:

- **50 or more employees**, OR
- **US$ 5 million or more** in annual revenue

This applies even if individual users within the organization use Veesker for personal projects on company devices — the licensing trigger is the **organization's size**, not the individual.

## What about modifying / redistributing the source?

The Apache License 2.0 governs the source code. You may:
- ✅ Read, modify, and study the source code freely
- ✅ Build your own version from source for any purpose (subject to Apache 2.0 terms)
- ✅ Contribute changes back via pull requests
- ✅ Fork the project under Apache 2.0

The commercial subscription requirement applies to the **packaged application** (installer downloads, official binaries) — not to the source code itself. If you build from source, that build is yours under Apache 2.0 with no warranty.

## Honor system

Compliance with this policy is the responsibility of the using organization. Veesker does not technically enforce the size threshold — there is no telemetry, no license server check, no degraded experience for non-subscribers. The application works the same regardless.

This is the same approach Docker Desktop uses. Compliance is enforced through:
- The end-user license agreement (EULA) signed during subscription
- Auditing rights reserved by the maintainer (rarely exercised, requires reasonable notice)
- Good faith from the user organization

## Commercial subscription includes

What you actually get when you pay:

- **Right to use** the official packaged Veesker desktop application commercially
- **Email and/or chat support** with response-time SLA per your tier
- **Optional: priority bug fixes** for issues you report
- **Optional: training and onboarding** sessions for your team
- **Optional: paid add-on packs** (e.g., Oracle EBS pack, Forms converter, on-prem AI gateway)

Higher tiers (Business, Enterprise) include access to **Veesker Cloud** (when launched) for team workspace sync, audit log shipping, and SSO integration.

## Why this model?

- **The source stays free and open** under Apache 2.0 — anyone can read, modify, contribute, and build from source
- **Individuals and small businesses get full access for free** — this is who tends to adopt new tools and drive word-of-mouth
- **Larger organizations contribute financially** to sustain development — this is fair given they extract more economic value from the tool
- **No technical lock-in** — if you leave, you keep your data (everything is on your machine), and you can keep using a self-built version

This model funds open-source development without compromising the open-source nature of the project.
